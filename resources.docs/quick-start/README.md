# G4 Quick Start — From Zero to Your First Automation

Welcome! 👋

This quick start takes you from a **completely empty machine** to **watching your own automation run** — and then to **recording one just by clicking around your app**. No prior G4 experience needed. Every step is written out click by click, with pictures.

Plan for about **30–40 minutes** the first time through.

---

## Who is this for?

Anyone who wants to get G4 running and build something real fast — analysts, testers, developers, or the simply curious. If you can run a command and use a file explorer, you can finish this.

## What you'll do

By the end you will have:

- Deployed a full **G4 sandbox** on your own machine with a single command
- Started working in the bundled VS Code, with the engine and recorders launching on demand
- Installed the **G4 VS Code extension** and created your first project
- Fetched a free **G4 token**
- Built and run a real automation: **open a page → wait → close the browser**
- **Recorded** your own automation just by interacting with an application

## Prerequisites

- A Windows or Linux machine with internet access
- Permission to run an install script (administrator/sudo may be requested by the installer)
- That's it. Everything else — including VS Code — comes bundled inside the sandbox.

> **💡 Windows vs Linux:** Screenshots and the main text use **Windows** (paths like `C:\g4-sandbox\...`). Where Linux differs, you'll see a note — Linux installs under `/opt` instead of `C:\`.
